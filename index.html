📖 How to Use Instant Indexer
⚠️ Important: You cannot directly download a JSON key from Search Console. You must create a Google Cloud Service Account, generate a JSON key from Google Cloud Console, and then add the service account email as an Owner in Search Console. Follow these steps carefully in order.
1
Create a Google Cloud Project
Go to console.cloud.google.com → Click on "Select a project" at the top → Click "New Project" → Enter any project name → Click Create.
2
Enable Google Indexing API
From the left menu → APIs & Services → Library → Search for "Web Search Indexing API" → Click on it → Click Enable.

↗ Enable via Direct Link
3
Create a Service Account
From the left menu → APIs & Services → Credentials → Click "+ Create Credentials" → Select "Service Account" → Enter a name (e.g., indexer-bot) → Click Done.

Next, click on the newly created Service Account → Go to the Keys tab → Click "Add Key → Create New Key" → Choose JSON format → Click Create.

✅ A .json file will automatically download to your computer. This is your service account key file.
4
Add Service Account as Owner in Search Console
This is the most important step! If you don't do this, indexing won't work.

1. Copy the "client_email" value from your .json file.
(Example: indexer-bot@your-project.iam.gserviceaccount.com)

2. Go to search.google.com/search-console and select the website you want to index.

3. Left menu → Settings → Users and permissions → Click "Add User" → Paste the copied client_email → Set Permission: Owner → Click Add.

↗ Search Console Users Settings
5
Upload JSON Key and Submit URLs
1. Come back to this tool → In the JSON Key Manager, click "Click to upload" and select your JSON file.

2. Paste the URLs you want to index in the URL box (one URL per line).

3. Select Action: URL_UPDATED (to index) or URL_DELETED (to remove from index).

4. Click ⚡ Submit All URLs.

💡 Daily Quota: Each JSON key allows a maximum of 200 URLs per day. To submit more URLs, add multiple keys (up to max 10).
🔴 What does the Red Mark mean? When a JSON key's daily quota of 200 URLs is exhausted, a red mark appears next to it. The mark will automatically turn green after 24 hours (when the daily quota resets).

What is Instant Indexer? — Free Google URL Indexing Tool

Instant Indexer is a free, open-source, browser-based tool that leverages the Google Indexing API v3 to submit URLs directly to Google Search for immediate crawling and indexing. Unlike the traditional Google Search Console "Request Indexing" button — which is slow and limited — this tool lets you submit up to 200 URLs per service account per day, instantly.

No backend server, no installation, no registration required. Everything runs in your browser using the Web Crypto API for secure JWT authentication with Google's OAuth 2.0.

Frequently Asked Questions

How do I get a Google service account JSON key for the Indexing API?
Go to Google Cloud Console, create a new project, enable the Indexing API, navigate to IAM & Admin → Service Accounts, create a service account, generate a JSON key, and download it. Then add that service account email as an Owner in your Google Search Console property.
Does Instant Indexer work for all websites, including WordPress and Blogger?
Yes. As long as you verify your website in Google Search Console and add the service account as an owner, you can use Instant Indexer to submit URLs from any website — WordPress, Blogger, custom HTML sites, or any other platform.
Is the Google Indexing API officially supported for all URLs?
Google officially supports the Indexing API for JobPosting and BroadcastEvent structured content. However, many SEOs and webmasters use it to request crawling of any URL. While it may trigger faster crawling, Google still decides what to index. Results may vary.
Can I use multiple service account keys to index more than 200 URLs per day?
Yes. Each Google Cloud service account key has a quota of 200 URL submissions per day. By uploading multiple keys (from different GCP projects), you can multiply your daily quota. Instant Indexer automatically rotates between keys to use available quota.
